# 角色设定

你是一位资深的设计系统架构师。我们要开发一款 [产品类型，如：SaaS 数据分析平台 / 冥想 App]。

# 任务目标

请帮我初始化一套完整的 Design Token 系统。

# 产品基调

1. 风格关键词：[例如：专业、冷静、科技感 / 温暖、圆润、亲和力]
2. 核心品牌色倾向：[例如：深蓝色系 / 翠绿色系]
3. 目标用户：[例如：企业财务人员 / 年轻学生]

# 执行步骤

1. **推导色板**：基于品牌色倾向，生成一套包含 Primary, Secondary, Success, Warning, Error, Neutral 的完整色板（50-900 阶梯）。
2. **定义排版**：设定基于 1.25 或 1.333 比例尺的字体大小系统（xs 到 2xl）。
3. **定义间距**：建立基于 4px 或 8px 网格的间距系统。
4. **定义圆角与阴影**：根据产品风格（锐利或圆润）定义半径和阴影层级。
5. **输出文件**：
   - 生成 `tokens.json`：作为单一数据源（Single Source of Truth），包含所有全局 tokens 和语义化 aliases。
   - 生成 `design-system.md`：一份人类可读的设计规范文档，解释每个 token 的用途。

# 约束条件

- 命名遵循 BEM 或 Kebab-case 规范（如 `color-brand-primary`）。
- 颜色必须考虑无障碍对比度（WCAG AA 标准）。
- 所有数值必须是逻辑单位，不要硬编码具体场景。
